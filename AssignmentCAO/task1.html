What is Docker?

Docker is an open platform for developing, shipping, and running applications. Docker enables you to separate your applications from your infrastructure so you can deliver software quickly. With Docker, you can manage your infrastructure in the same ways you manage your applications. By taking advantage of Dockerâ€™s methodologies for shipping, testing, and deploying code quickly, you can significantly reduce the delay between writing code and running it in production.

My OS is based on predefined Docker Image:

randomdude/gcc-cross-x86_64-elf

We can run an instance of that image (known as container) using :-

docker run --rm -it -v "%cd%":/root/env myos-buildenv

Bootloader locates the operating system using the magic number:

; magic number dd 0xe85250d6 ; multiboot2

main.asm

global start
section .text
bits 32
start:
; print `OK`
//writing directly to video memory
mov dword [0xb8000], 0x2f4b2f4f
//freezs all other operations
hlt //freezs all other operations

linker.Id file defines how the OS is linked and start(which is defined in main.asm) is defined as the Entry point.

What are MakeFiles?

Make is a build automation tool that automatically builds executable programs and libraries from source code by reading files called Makefiles which specify how to derive the target program

Finally emulate the operating system using QEMU:

qemu-system-x86_64 -cdrom dist/x86_64/kernel.iso
